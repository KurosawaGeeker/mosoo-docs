## 第一个请求

这个请求会为已发布 Agent 启动一项新的工作。把 token 和 Agent ID 换成你自己的值。

```bash
curl -X POST "https://mosoo.ai/api/v1/agents/$MOSOO_AGENT_ID/threads" \
  -H "Authorization: Bearer $MOSOO_API_TOKEN" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: ticket-182-create-thread" \
  -d '{
    "client_external_ref": "ticket-182",
    "input": {
      "type": "user.message",
      "content": [
        {
          "type": "text",
          "text": "Triage this customer escalation and suggest next steps."
        }
      ]
    }
  }'
```

响应里会包含 `thread.id`。请保存它。后续发送消息、附加文件、读取 Agent 回复、归档这项工作，都要使用同一个 Thread ID。

## 可以用来做什么

当 Agent 的工作需要从 mosoo 之外开始时，可以使用 API：

- 工单创建后，让 Agent 先做分诊。
- 订单、告警或定时任务出现时，自动启动 Agent。
- 在 Agent 开始前附加 brief、截图、日志或客户文件。
- 在你的产品里展示 Agent 的回复和处理进度。
- 用 `client_external_ref` 把 mosoo Thread 和你自己的记录对应起来。

## 访问规则

每个 API 请求都要通过两个检查：

- Agent 已发布，并且启用了 API 接入。
- token 有效，并且被允许调用这个 Agent。

token 不会自带新的模型设置或渠道凭据。Agent 仍然使用它在 mosoo 中由所有者配置好的能力。
