# 基于已发布 Mosoo Agent 开发

Mosoo API 用于在你的应用中调用**已发布**的 Mosoo Agent。你的应用提供调用方 token 和已发布 Agent 的 `agentId`；Mosoo 会创建 Thread、启动 Agent，并向 API 调用方开放这个 Thread 的公开事件和文件。

这就是 Agent 发布菜单里 **API 接入（API Access）** 入口对应的集成能力。

<Note>
  API 只适用于已发布的 Agent。Agent 配置、模型服务商连接和渠道绑定都需要先在 Mosoo 中完成。
</Note>

## 第一个请求

把 access token 和 Agent ID 组合起来，就可以创建 Thread。

```bash
curl -X POST "https://mosoo.ai/api/v1/agents/$MOSOO_AGENT_ID/threads" \
  -H "Authorization: Bearer $MOSOO_API_TOKEN" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: ticket-182-create-thread" \
  -d '{
    "client_external_ref": "ticket-182",
    "input": {
      "type": "user.message",
      "content": [
        {
          "type": "text",
          "text": "Triage this customer escalation and suggest next steps."
        }
      ]
    }
  }'
```

响应里会包含 `thread.id` 和第一个 `run`。后续发送消息、读取事件、附加文件和管理生命周期都使用这个 Thread ID。

## 可以用来做什么

Mosoo API 让你把已经发布的 Agent 接入自己的应用。你可以把 Mosoo 托管的 Agent 当成一个可调用的后端：你的系统发送输入，Agent 在 Mosoo 的运行环境和工作空间中处理任务，然后你的应用读取回复、状态和文件结果。

你可以用它来：

- 做自己的聊天机器人、移动 App 或网页应用，后端调用 Mosoo Agent。
- 给定制应用接入一个已经配置好的 Agent，让它处理用户请求、文件和上下文。
- 用代码协调多个 Agent，把它们组合成一个协作工作台或自动化流程。
- 用定时任务、外部事件或自己的服务触发这个常驻在线的 Agent，让它按需处理某类工作。

## 它怎么工作

1. 先在 Mosoo 中创建并发布 Agent。
2. 在你的应用后端保存 API token 和 Agent ID。
3. 通过 API 创建 Thread，把用户输入或文件交给 Agent。
4. 读取 Thread 的回复、进度、状态和文件结果。
5. 需要继续对话或继续任务时，向同一个 Thread 发送新的输入。

## 访问模型

每个请求都会同时校验已发布 Agent 和调用方 token：

- Agent 必须存在、已发布，并启用 API 接入。
- 调用方 token 必须有效且未被撤销。
- 调用方仍需通过组织成员和 Agent 访问检查。
- 运行时会使用已发布 Agent 所有者配置的能力边界。
