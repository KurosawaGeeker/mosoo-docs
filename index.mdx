# Groots API

The Groots Published Agent API lets approved callers submit tasks to published Agents.

This is the small public docs surface for M2D:

- [Quickstart](/quickstart): create a task, check its status, and continue with the Session API when needed.
- [Authentication and access](/auth-and-access): understand Human PAT callers, Organization Service tokens, attribution, Agent access, and owner execution.
- API Reference: generated from the live OpenAPI document.

The default entry point is `POST /api/v1/agents/{agentId}/tasks`. Creating a task creates the backing Agent Session and the initial run. Advanced lifecycle controls, including sending follow-up events, retrieving session metadata, session files, archive, unarchive, and delete, stay on the lower-level Session API.

The API is bearer-token authenticated and access-gated. A Human Personal Access Token identifies a human caller. An Organization Service token identifies a machine caller and can optionally project attribution to a human user when allowed. Runtime execution still follows the published Agent owner's configured access boundary.
